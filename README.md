# EduAI — Render Deployment Guide

## Quick Deploy to Render

### Step 1 — Push to GitHub
```bash
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/YOUR_USERNAME/eduai.git
git push -u origin main
```

### Step 2 — Create Web Service on Render
1. Go to https://render.com → **New → Web Service**
2. Connect your GitHub repository
3. Render will auto-detect `render.yaml` — click **Apply**

### Step 3 — Set Secret Environment Variables
In Render Dashboard → your service → **Environment**, add:

| Key | Value |
|-----|-------|
| `PINECONE_API_KEY` | Your Pinecone API key |
| `GEMINI_API_KEY` | Your Gemini API key |
| `PINECONE_INDEX_NAME` | eduai-students (or your index name) |

> SECRET_KEY is auto-generated by Render via render.yaml.

### Step 4 — Deploy
Click **Deploy**. Build takes ~2-3 minutes.

---

## Environment Variables Reference

| Variable | Description | Required |
|----------|-------------|----------|
| `SECRET_KEY` | Flask session secret (auto-generated) | Yes |
| `DATABASE_URL` | SQLite by default. Use PostgreSQL for production persistence | Yes |
| `PINECONE_API_KEY` | From https://app.pinecone.io | Yes |
| `PINECONE_INDEX_NAME` | Your Pinecone index name | Yes |
| `GEMINI_API_KEY` | From https://aistudio.google.com | Yes |

---

## Important: Persistent Storage

Render's free tier has an ephemeral filesystem — uploaded files and SQLite data
are wiped on each deploy/restart.

For production use:
- Use Render PostgreSQL (add-on) for the database
- Use Cloudinary / S3 for uploaded PDFs/images

To switch to PostgreSQL, set DATABASE_URL to your Render Postgres connection string.
SQLAlchemy will handle it automatically — no code changes needed.
